## 🔜 Текущая очередь

### 1. Облачный бэктест-фермер (24/7 массовое тестирование) 🆕
_Цель: автоматизировать массовый прогон стратегий по инструментам из скринера с учётом фаз рынка, параметров риск-менеджмента, и получать промежуточные результаты. Сессии клиента не влияют на выполнение – тесты продолжаются на сервере, а при повторном подключении клиент забирает все пропущенные результаты._

#### 1.1. Перенос задач в постоянное хранилище (Supabase)
- [x] Создание таблицы `backtest_tasks` (уже выполнено ранее, но нужно дополнить полями `user_id`, `batch_id`, `status`, `result`, `error`)
- [x] **Расширение схемы** – добавить таблицу `backtest_batches` для группировки задач одного прогона:
  - `id` (batch_id)
  - `user_id`
  - `created_at`
  - `params` (json – общие настройки)
  - `status` (pending/running/completed/failed)
- [x] **Изменение BacktestQueue** – при старте задачи сразу сохранять запись в Supabase, обновлять статус и результат по завершении.

#### 1.2. Расширение серверного API
- [x] **`POST /api/backtest/batch`** – принимает массив инструментов и общие параметры (стратегия, риск-менеджмент, диапазоны дат). Создаёт batch-запись и для каждого инструмента ставит отдельную задачу в очередь. Возвращает `batchId`.
- [x] **`GET /api/backtest/batch/:batchId`** – возвращает статус всего batch'а и список подзадач с их статусами.
- [x] **`GET /api/backtest/batch/:batchId/results`** – отдаёт агрегированные результаты (статистику и сделки по всем задачам batch'а) в формате JSON/CSV.
- [ ] **SSE-эндпоинт `/api/backtest/progress/:batchId`** – стримит обновления статусов задач в реальном времени (для активного наблюдения). При отключении клиента прогресс сохраняется в Supabase и доступен через polling.
- [ ] **`GET /api/backtest/tasks`** (уже есть) – дополняется фильтром по `batchId`, чтобы клиент мог запрашивать только свои задачи из конкретного прогона.

#### 1.3. Клиентская интеграция (Electron)
- [ ] **UI «Облачный фермер»** – новая вкладка, где можно:
  - Выбрать инструменты (из скринера или вручную)
  - Задать диапазоны дат, параметры стратегий и риск-менеджмента
  - Запустить batch на сервере
  - Видеть прогресс выполнения всех подзадач (live через SSE или polling)
  - Сохранять/загружать конфигурации batch'ей
- [ ] **Автономность** – после запуска batch'а клиент может отключиться; при повторном подключении он получает список своих активных/завершённых batch'ей и может скачать результаты.
