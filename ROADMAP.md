- [x] **Диагностика ошибки 30028** — найден корень проблемы (требуется UUID в `orderId`)
- [x] **Хелпер `generateUUID()` в `OrderManager`** — использует `crypto.randomUUID()`
- [x] **Исправление `placeStopOrders`** — генерация UUID для `orderId`, корректные цены
- [x] **Сохранение `placeProtectiveOrders`** — метод с лимитными ордерами как альтернатива
- [ ] **Конфигурация `stopMode` и UI‑переключатель** — выбрать `stop_order` или `limit_order`
- [ ] **Интеграция в `processSignal`** — вызов нужного метода в зависимости от `stopMode`
- [ ] **Совместимость с трейлинг‑стопом** — работает для обоих режимов через `replaceSandboxOrder`
- [ ] **Тестирование** — стоп‑заявки с UUID проходят, лимитные ордера выставляются
